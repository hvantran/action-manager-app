# Environment Setup with direnv

This project uses [direnv](https://direnv.net/) to automatically manage the Node.js environment.

## What is direnv?

direnv is an environment switcher for the shell. It loads environment variables depending on the current directory, making it perfect for managing project-specific Node.js versions.

## Installation

### macOS
```bash
brew install direnv
```

### Ubuntu/Debian
```bash
apt install direnv
```

### Configuration

Add to your shell configuration file (`~/.bashrc`, `~/.zshrc`, etc.):

```bash
# For bash
eval "$(direnv hook bash)"

# For zsh
eval "$(direnv hook zsh)"
```

Then reload your shell:
```bash
source ~/.bashrc  # or ~/.zshrc
```

## Usage

### First Time Setup

When you first enter the project directory, direnv will detect the `.envrc` file but block it for security:

```bash
cd action-manager-ui
# direnv: error .envrc is blocked. Run `direnv allow` to approve its content
```

Review the `.envrc` file and approve it:
```bash
direnv allow
```

### Automatic Version Switching

Once approved, direnv will automatically:
1. Load nvm when you enter the directory
2. Switch to Node.js 20.18.1 (as specified in `.nvmrc`)
3. Display the current Node version

```bash
$ cd action-manager-ui
Now using node v20.18.1 (npm v11.1.0)
Using Node.js version: v20.18.1
```

### Manual Version Switching (without direnv)

If you don't use direnv, manually switch Node versions using nvm:
```bash
nvm use
# Or
nvm use 20.18.1
```

## Files

- `.nvmrc` - Specifies Node.js version (20.18.1)
- `.envrc` - Direnv configuration that loads nvm and switches Node version
- `.envrc.md` - This documentation

## Benefits

- ✅ Automatic Node version switching per project
- ✅ Ensures all team members use the same Node version
- ✅ Prevents "works on my machine" issues
- ✅ No manual `nvm use` commands needed

## Security

Always review `.envrc` files before running `direnv allow`. The `.envrc` file can execute arbitrary shell commands, so only approve trusted files.
